What It Does

SnapCase turns public pages into organized case evidence.

The app is designed around the investigator's working path: preserve the source, save the local evidence, extract useful signals, and keep everything tied to the case.

Preserve the source

Capture public profiles, posts, comments, media, screenshots, source URLs, local files, and evidence logs while you review the page.

Understand what matters

Run OCR, keyword search, target checks, face and object matching, account context checks, and bookmark review from the case workspace.

Report the case

Create HTML reports with source links, local links, hashes, comments, thumbnails, activity logs, and investigator details.

Automated Features

Automation handles the repetitive evidence work.

SnapCase automation is built to save investigator time without hiding what happened. Outputs stay tied to thumbnails, source links, local files, and case activity.

01

Background OCR and keyword checks

Images, screenshots, bookmarks, comments, and sampled video frames can be processed for searchable text.

Example: flag a name, phone number, handle, or phrase that appears inside a screenshot.
02

Bookmark enrichment

Saved findings can be enriched with thumbnails, local media links, source context, capture files, and report references.

Example: save a post now, then review the local media and OCR result later.
03

Target alerts

Case targets help identify matching names, handles, keywords, faces, objects, or other recurring case signals.

Example: alert when a known handle appears in a new captured report.
04

Report generation

Platform reports and global bookmark reports keep evidence readable for review, handoff, and disclosure preparation.

Example: export a focused report for all selected items in a case.
05

Case activity logs

Captures and analysis steps can be recorded with timestamps, source context, local paths, and investigator details.

Example: show when a source was captured and where the local evidence was saved.
06

Processing queues

Longer-running work can be queued so the investigator can keep organizing the case while analysis continues.

Example: queue OCR and media checks after a profile or post capture.

Built-In Intelligence

SnapCase helps connect evidence without turning it into a black box.

The intelligence layer is meant to make leads explainable: thumbnails, source links, local links, confidence, and context stay visible.

Cross-index

Find repeated identifiers, media, names, handles, and visual targets across the local encrypted case index.

Example: the same username appears in two unrelated cases.

OCR

Extract text from captured images, screenshots, bookmark media, reports, and sampled video frames.

Example: a phone number appears in a photo, not in page text.

Facial recognition

Build a face index for case media and compare selected faces against captured evidence.

Example: a person appears across multiple public posts under different accounts.

User index cards

Keep user details, aliases, profile links, thumbnails, source notes, and case observations in a reviewable card.

Example: one card collects public profile context and related evidence hits.

Why It Fits Your Company

Choose local control, team sharing, server storage, or managed AWS.

SnapCase can be positioned for teams that need evidence to stay inside the office, or for teams that want the heavy processing moved to managed cloud infrastructure.

Local workstation

Best when the investigator wants public evidence, reports, OCR, and local case files to stay on their machine.

Small Team / NAS

Best when a small office wants a shared master index and repeated identifier/media discovery across team cases without cloud storage.

Corporate Server

Best when a company wants an on-prem file keeper and controlled shared evidence repository behind its own policies.

AWS Managed

Best when a team wants to offload OCR, image/video processing, storage, retention, audit logs, and support to powerful AWS infrastructure.

If you do not want data leaving the office, SnapCase can be sold around local, NAS, and on-prem server options. If you want to offload the work, the AWS option moves heavy processing and storage away from investigator machines.
